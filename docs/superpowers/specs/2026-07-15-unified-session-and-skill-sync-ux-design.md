# 统一会话与 Skills 同步体验设计

## 目标

- 云端会话卡片直接比较本地与云端内容，不再依赖抽象标签猜测状态。
- 从远程会话预览返回时回到原列表，并保留搜索、筛选、选择和滚动位置。
- Skills 批量操作只保留一套真实进度状态，结束后一定展示最终结果。
- 每个逻辑 Skill 在列表中只出现一次，详情内统一展示 Local、Remote 与 Diff。

## 云端会话清单

### 卡片结构

每行由标题区、双列比较区和操作区组成：

```text
会话标题  来源                                      待更新云端
项目路径 · 分支

本地                         云端
4 分钟前                     8 小时前
41 条消息                    39 条消息
                             同步于 8 小时前

                                                        上传
```

- “本地”和“云端”使用相同的中性色文字与设备/云图标，不再各自使用容易混淆的彩色胶囊标签。
- 每侧分别展示该副本的内容更新时间和消息数，不使用两端混合后的最大值。
- 缺失的一侧明确显示“未上传”或“无本地副本”。
- 标题右侧只保留一个同步关系标签：仅本地、待更新云端、已同步、云端较新、仅云端、内容冲突。
- 颜色只表达同步关系：已同步为绿色，待上传/云端较新为强调色，冲突为警告色；本地和云端位置本身不使用状态色。

### 操作

- 每行只显示一个与当前状态匹配的主操作：
  - 仅本地、待更新云端：上传。
  - 仅云端、云端较新：恢复为新的本地副本。
  - 内容冲突：处理冲突，提供“覆盖云端”和“把云端恢复为新副本”两种明确选择。
  - 已同步：查看。
- 查看详情、恢复到来源 SSH 环境和删除云端副本放入 `…` 次要菜单；删除确认继续明确本地内容不会删除。

### 预览返回

- 远程会话列表与预览形成同一导航栈。打开预览时列表保持挂载，预览覆盖在其上方。
- 关闭按钮和 Escape 都先关闭预览，返回原远程会话列表；再次关闭才回主界面。
- 返回后保留查询、来源筛选、已选行和滚动位置。

## 统一 Skills 清单

### 列表模型

- 删除顶部用于拆分两个数据集的 Local / Remote 大标签。
- 根据可移植 identity 与已有同步关系，把本地 Skill 和云端 Skill 合并成一个逻辑条目；云端独有、本地独有也各显示一条。
- Plugin、System、Project Skill 仍是独立本地条目，不尝试与云端记录自动合并。
- 列表项首行固定为“名称 + 来源标签 + 同步状态”。来源标签紧跟名称，名称占剩余空间并在过长时省略，悬停和详情显示完整名称。
- 不再使用相同蓝色的 `v1` / `Local v1` 胶囊表示两个位置。列表只显示简洁同步状态，具体版本进入详情比较。
- 搜索、来源筛选和排序作用于统一清单。未配置云端时也能正常查看全部本地 Skills。

### 批量操作

- 复选框只出现在可同步的用户级和共享 Skill 上。
- 选中后显示一条上下文操作栏，根据实际可执行项给出“上传到云端”“更新本地”“删除云端副本”，按钮显示可操作数量。
- Plugin、System、Project Skill 不进入云端批量操作。
- 所有批量操作只使用一份状态：`运行中（当前/总数）`、`成功`、`部分失败`或`失败`。
- 完成后运行中文案必须被最终汇总替换；成功结果可自动消失，冲突和失败保持显示并保留失败项选择。

## Skill 详情

每个逻辑 Skill 的右侧详情固定提供 `Local / Remote / Diff`：

### Local

- 展示本地完整名称、来源、安装路径、更新时间、使用次数和本地内容。
- 本地不存在时显示“尚未安装”，并在云端可用时提供安装入口。
- 来源标签紧跟名称；长名称省略但不挤压标签。

### Remote

- 健康状态正常时展示云端版本历史、所选版本更新时间、远程内容以及下载/删除操作。
- 未配置 URL/key 时，只有用户主动进入 Remote 后才显示配置说明。
- 缺表、缺列、bucket 或权限时显示复制最新 SQL、打开 SQL Editor 和刷新。
- 鉴权或网络错误显示检查 URL/key 的说明，不错误引导执行 SQL。
- Plugin、System、Project Skill 显示各自管理方式，不显示上传或安装到猜测目录的入口。

### Diff

- 本地和云端都存在时，先展示内容 hash 关系和版本摘要。
- 展示文件清单差异：新增、删除、修改、未变化；默认展开 `SKILL.md` 的统一文本差异。
- 其他文本文件可点开查看差异；二进制文件只显示大小和 hash 是否变化。
- 任一侧不存在时，Diff 显示“仅本地”或“仅云端”的结构化摘要，不伪造逐行差异。
- 冲突状态在 Diff 页提供两个明确操作：上传并覆盖云端，或把云端恢复为新的本地副本。

## 状态与身份

- 继续使用六种同步关系：仅本地、本地较新、已同步、云端较新、仅云端、冲突。
- Skill 合并优先使用 `portableScope + relativePath`；旧版无法安全定位的记录保持独立并标记为旧版记录。
- 不用修改时间决定覆盖方向；内容 hash 与上次绑定 revision 决定同步状态。

## 错误处理

- 会话或 Skill 单项操作失败时不清空其他成功结果。
- 批量失败项保持选中，用户可以直接重试。
- 预览加载失败停留在远程会话列表并显示错误，不跳回主界面。
- Remote 详情的修复操作完成后原地刷新，不改变当前 Skill、版本或详情页签。

## 测试与验收

- 六种会话状态都验证双列内容、缺失侧文案和唯一主操作。
- 验证远程预览的关闭按钮与 Escape 都返回原列表并保留列表状态。
- 验证批量上传从运行中切换到最终汇总，不残留“正在上传”。
- 验证统一 Skill 列表能合并匹配项、保留本地独有与云端独有，并隔离同名不同 identity。
- 验证来源标签紧跟名称、长名称省略且标签不被挤压。
- 验证 Remote 页在未配置、需要 SQL、鉴权失败和健康状态下分别显示正确内容。
- 验证 Diff 对 SKILL.md、附带文本文件、二进制文件和单边缺失的结果。
- 所有测试使用临时目录与合成 Skill/会话，不读取或修改真实用户数据。
