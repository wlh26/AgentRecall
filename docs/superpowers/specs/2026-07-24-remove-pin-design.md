# 移除会话置顶功能

## 背景

AgentRecall 目前把“置顶”作为一项独立的会话整理能力。入口分布在会话列表、详情菜单、可见性筛选和 MCP 写入工具中；底层还会用置顶状态调整默认排序与搜索相关性。

本次从 `main` 移除这项能力。收藏、隐藏、标签和自定义标题不受影响。

## 方案

删除所有用户可触达的置顶行为，以及依赖置顶状态的排序和搜索逻辑。现有 SQLite 数据库中的 `pinned` 列暂时保留，但应用不再读取它来决定展示、排序或搜索结果，也不再提供写入入口。

暂时保留数据库列是为了让已有安装直接升级，不引入重建 `sessions` 表的高风险迁移。该列只承担旧数据库兼容作用，不再属于产品能力。

## 改动范围

### 界面

- 删除“置顶”可见性筛选。
- 删除会话列表和详情中的置顶、取消置顶操作。
- 删除会话行上的置顶图标。
- 查询构建器只保留全部、收藏和隐藏三种可见性。

### 应用接口

- 删除 renderer 到 main process 的 `setPinned` preload API。
- 删除 `pin:set` IPC handler。
- 删除会话存储对外暴露的 `setPinned` 方法。
- MCP 的 `set_visibility` 不再接受 `pinned`。`default` 只恢复正常可见状态，不再修改旧置顶字段。

### 查询与同步

- 默认列表不再把旧置顶会话排在前面。
- 关键词搜索和智能排序不再给旧置顶会话加权。
- 删除 `pinned` 可见性查询。
- 会话合并与远程同步不再传播置顶状态。已有数据库值会被忽略，不影响收藏、隐藏和其他元数据。

### 数据兼容

- 不删除 SQLite 的 `pinned` 列，也不重建表。
- 新代码不提供写入入口，不以该字段决定任何用户可见行为。
- 后续如果需要清理物理字段，应单独设计数据库迁移，不与本次产品能力移除混在一起。

### 文档与发布说明

- 从中英文 README 和安装说明中删除置顶能力描述。
- 本分支只添加一份 release note，说明会话整理入口中不再提供置顶功能。
- release note 使用 `## Bug 修复`，避免把能力移除描述成新增功能。

## 测试

测试以可观察行为为准：

- UI 契约中不再出现置顶筛选、按钮和图标。
- 查询构建器类型不接受 `pinned`。
- MCP 拒绝 `pinned`，收藏、隐藏和恢复默认状态仍能正常工作。
- 人工写入旧 `pinned` 值后，默认排序、空查询排序和搜索相关性都不发生变化。
- 会话重建索引、远程同步和迁移继续保留其他元数据，不再断言置顶状态。
- 完成实现后运行相关测试、完整测试、类型检查、构建和 `npm run release-note:check`。

## 不在本次范围内

- 不删除或重建已有数据库表。
- 不修改收藏、隐藏、标签和自定义标题的交互。
- 不调整与置顶无关的搜索排序规则。
- 不处理注释中“固定到视口顶部”这类布局语义；它们不是会话置顶功能。
